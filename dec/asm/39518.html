39518: Hit range validation and combat event dispatch.
Used by the routine at hit_detection.
Called by hit_detection when attacker and defender frame states match a hit table entry. Validates the hit by comparing fighter_distance fighter distance against the four packed range values in byte 0 of the hit_type_table indexed by hit type. If no range matches, the hit is silently ignored.
On a valid hit, byte 1 gives the defender reaction movement ID (255 = no change), applied via force_apply_moves. Byte 2 high nibble gives the combat event type stored in pending_sound_index. Byte 2 low nibble gives the knockback type passed
to energy_dec to decrement energy.
INPUT: HL = pointer into hit section pair (defender frame ID, hit type index). IX = attacker state block, IY defender state block.
OUTPUT: None. DESTROYS: AF, BC, DE, HL.
