39020: X position update and tackle proximity gate.
Used by the routine at char_frame_update.
Advances the active character X position tile via x_pos_controller, then recomputes the fighter distance. If the new distance is negative (bit 7 set), the fighters have overlapped after the step. In that case, if neither fighter is in an exit or death movement, tackle detection is armed (tackle_overlap_flag=1). If one fighter is rolling (mv17) and the other has reached the jump-over frames (8 or 9), the flag stays set and tackle_side_swap handles the cross-side sequence. If no roll/jump-over match is found, the step is reversed.
INPUT: IY = active character state block. B = X tile before the step (saved by char_frame_update).
OUTOUT: tackle_overlap_flag set to 1 if tackle condition is met; 0 if the step was rolled back. DESTROYS: AF, BC.
