35878: Keyboard scanner: detects single pressed key for input config (redefine keys).
Used by the routine at catch_key.
Scans 8 keyboard rows (port 0xFE) to find exactly one keypress. Used by key assignment (catch_key) to capture scancodes without conflicts.
OUT: D=scancode (0-39) or 255 (none/multiple); Z=0 if valid key.
DESTROYS: AF,BC,DE,H.
keyb_scan 35878 LD DE,65327 D = 255 (no-key sentinel); E = 111b (row/column base counter).
35881 LD BC,65278 B = 0xFE (keyboard row mask); C = 0xFE (ULA keyboard port).
keyb_scan_0 35884 IN A,(C) IN A,(C) read 5 key bits for this row from port 0xFE.
35886 CPL CPL invert bits (active-low): 1 = pressed, 0 = released.
35887 AND 31 AND 31 mask to bits 4-0 (the 5 keys of this row).
35889 JR Z,keyb_scan_2 If no key bits set in this row, jump to next row.
35891 INC D First key in this scan: INC D makes D = 0.
35892 RET NZ If D was already 0, a previous row already had a key -> two keys pressed; abort and return immediately with D=0, Z clear.
35893 LD H,A H = pressed key bitmask for this row (5 bits).
35894 LD A,E A = current row/column base (E).
keyb_scan_1 35895 SUB 8 Subtract 8 repeatedly to align A to the correct row base slot.
35897 SRL H SRL H shift key bitmask right; lowest set bit falls into carry.
35899 JR NC,keyb_scan_1 If carry clear, no key at this column; loop to check next column.
35901 RET NZ Single column found: return now; D already holds scancode.
35902 LD D,A D = A: store computed scancode (row base + column offset).
keyb_scan_2 35903 DEC E E- advance row/column base for next row.
35904 RLC B RLC B rotate row mask left to select next keyboard row.
35906 JR C,keyb_scan_0 If more rows remain (carry clear), loop back to read next row.
35908 CP A All rows scanned with no key found: CP A sets Z flag.
35909 RET Z Return with Z set and D = 255 (no key pressed or conflict).
