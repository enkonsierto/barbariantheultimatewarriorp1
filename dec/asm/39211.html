39211: Movement transition resolver.
Called from input_gated_moves with the mirror-adjusted key state in A.
Performs a two-level table lookup to find the candidate movement ID:
Level 1: maps the lower 4 direction bits of the key state to an input class (0-8) via input_lookup_table, collapsing 16 combos into 9 meaningful classes.
Level 2: uses the Fire button (IY+1 bit 4) to select the transition table - no_fire_transition_table (no fire: walks, crouches, rolls) or fire_transition_table (fire: attacks) - then resolves a pointer for the current movement ID and reads the resulting movement ID at the input class offset.
Returns 255 if no transition is defined for the current input/movement combination.
INPUT: A = adjusted key state. IY = character state block.
OUTPUT: A = new movement ID, or 255 if no transition applies. DESTROYS: AF, DE, HL.
