36363: Key assignment loop.
Used by the routines at one_player_mode and control_setup.
Draws the key-assignment screen using the string table at select_keys_player, then captures 5 action keys (Fire, Up, Down, Left, Right) storing each scancode into consecutive bytes of the IX table via catch_key.
Pause (pause_scancode) and Quit (quit_scancode) are captured conditionally: only if their buffer is 0.
After all keys are captured, waits ~1.57s (14 x 112ms) then fades to black via menu_fade_out and jumps back to the main menu loop at main_menu_0.
NOTE: The CALL at 36502 is temporarily patched to RET (0xC9) by two_player_mode option "4" so that P1 key capture returns here instead of jumping to main_menu_0, allowing a second call to control_setup for P2 key capture.
INPUT: IX = target scancode table (p1_keyboard_table P1 keyboard or p2_keyboard_table P2 keyboard).
OUTPUT: IX table filled with captured scancodes. pause_scancode and quit_scancode updated.
DESTROYS: AF, BC, DE, HL, IX.
