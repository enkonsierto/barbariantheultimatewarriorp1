39617: Main combat loop.
Used by the routine at entry_point.
Executed once per frame during active combat. Handles input, animation, rendering and death exit for both fighters and the goblin.
Frame processing order:
1. Compute horizontal fighter distance via calc_distance, stored in fighter_distance.
2. Dispatch input or AI for P2 then P1 via input_ai_dispatcher.
3. Update animation, movement and X position for P2 then P1 via char_frame_update.
4. Draw P1 and P2 sprites via sprite_dispatcher; Z-order determined by movement properties bit 0.
5. Draw the P2 overlay tile via p2_overlay_tile.
6. If the goblin (goblin_state_block) is inactive, return immediately. Otherwise advance and draw it via anim_frame_stepper and sprite_dispatcher, skipping the frame advance if the skip counter (frame_counter_hit) is non-zero.
7. Death check: if P1 holds movement 8 or 27, set P2 to movement 19 (victory) with IY=P1 as the dying character. Otherwise set P1 to movement 19 with IY=P2 as the dying character.
8. Goblin sync and exit: when the goblin's X tile reaches 29, deactivate the goblin and select exit animation 29, 31, 32 or 33 based on mirror flag and movement ID.
Movement IDs: 8 = decapitation death, 27 = standard death, 19 = victory pose.
INPUT: None. OUTPUT: None. Framebuffer, character state blocks and fighter_distance updated in place.
DESTROYS: AF, BC, DE, HL, IY, IX.
Frame processing: compute fighter distance, dispatch input or AI for both characters, update animation and X position, draw sprites in Z-order, draw P2 overlay tile, and advance and draw the goblin if active.
