35996: Single UDG character renderer
Used by the routines at hit_processor, catch_key, draw_word, print_dec_number, energy_dec and game_start.
Draws one 8x8 UDG tile from the font table at udg_font_addr to a VRAM address. Used to render scores, keys glyphs, menu labels and the hit indicator.
Converts the ASCII code to a font index (SUB 48), multiplies by 8 to get the byte offset, and adds the UDG font base to locate the 8-byte tile data. Copies one byte per scanline row, calling calc_next_scanline_addr after each row to advance to the next VRAM scanline.
INPUT: A = ASCII code (48+). HL = VRAM destination address. OUTPUT: None.
DESTROYS: AF, BC, DE, HL.
draw_character 35996 PUSH HL Preserves the VRAM destination address.
35997 SUB 48 Converts ASCII code to font index (index = A - 48).
35999 LD L,A Places font index in HL (L = index, H = 0).
36000 LD H,0
36002 ADD HL,HL Multiplies font index by 8 to obtain the byte offset into the font table.
36003 ADD HL,HL
36004 ADD HL,HL
36005 LD DE,23720 Loads the UDG font base address udg_font_addr into DE.
36008 ADD HL,DE HL now points to the first byte of the font data for this character.
36009 POP DE Restores the VRAM destination address into DE.
36010 EX DE,HL Swaps registers so HL = VRAM destination, DE = font source.
36011 LD B,8 Sets row counter to 8, one iteration per scanline.
draw_character_0 36013 LD A,(DE) Copies one font scanline byte to the screen.
36014 LD (HL),A
36015 CALL calc_next_scanline_addr Advances HL to the next VRAM scanline via calc_next_scanline_addr.
36018 INC DE Advances to the next font row.
36019 DJNZ draw_character_0 Repeats for all 8 scanlines.
36021 RET Return.
