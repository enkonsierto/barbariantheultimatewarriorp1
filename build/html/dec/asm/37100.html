37100: Apply edge mask and blend tile byte to framebuffer.
Used by the routines at left_mask_gen and right_mask_gen.
Shared endpoint for the left-edge mask generator (left_mask_gen) and the right-edge mask generator (right_mask_gen) when a sprite partially overlaps a tile column boundary.
The caller passes a pixel mask in A (1,3,7,15,31,63,127 or the MSB-packed equivalents for the left edge). The routine clears the transparent columns in the framebuffer byte and overlays the visible sprite columns from the tile byte.
INPUT: A = pixel mask. HL = framebuffer address. DE = tile source address of the byte to blend in.
OUTPUT None. Framebuffer byte at (HL) updated in place.
DESTROYS A, C.
mask_and_blend 37100 LD C,A save mask (A) into C before A is overwritten.
37101 LD A,(HL) A = framebuffer byte at HL.
37102 AND C Clear the transparent pixels in the background.
37103 LD (HL),A Write back to framebuffer the masked background.
37104 LD A,(DE) A = tile source byte at DE.
37105 XOR C Invert mask bits to keep the visible sprite pixels.
37106 LD C,A Now C contains the modified tile byte.
37107 LD A,(HL) A = framebuffer byte at HL (already masked).
37108 OR C OR C to overlay the sprite pixels.
37109 LD (HL),A Write blended result back to framebuffer.
37110 RET Return.
