901F: Calculates the VRAM address of the scanline below.
Given a VRAM scanline address in HL, steps forward one scanline, handling tile-row and screen-third boundaries in the non-linear ZX Spectrum screen layout.
INPUT: HL = current VRAM scanline address. OUTPUT: HL = address of the scanline below. DESTROYS: A, F.
calc_next_scanline_addr 901F INC H Steps forward one scanline (H encodes the scanline index within the tile).
9020 LD A,H Checks whether the increment crossed a tile boundary (lower 3 bits of H become 0 when wrapping forward).
9021 AND $07
9023 RET NZ Within the same tile; address is ready.
9024 LD A,L Moves down one tile row (L encodes the tile row; add 32 to reach the row below).
9025 ADD A,$20
9027 LD L,A
9028 RET C Tile-row overflow carried into the screen-third; address is ready.
9029 LD A,H No carry; the screen-third was over-incremented - subtracts 8 from H to correct it.
902A SUB $08
902C LD H,A
902D RET Return.
