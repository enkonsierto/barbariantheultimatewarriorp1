9AC1: Main combat loop.
Used by the routine at entry_point.
Executed once per frame during active combat. Handles input, animation, rendering and death exit for both fighters and the goblin.
Frame processing order:
1. Compute horizontal fighter distance via calc_distance, stored in fighter_distance.
2. Dispatch input or AI for P2 then P1 via input_ai_dispatcher.
3. Update animation, movement and X position for P2 then P1 via char_frame_update.
4. Draw P1 and P2 sprites via sprite_dispatcher; Z-order determined by movement properties bit 0.
5. Draw the P2 overlay tile via p2_overlay_tile.
6. If the goblin (goblin_state_block) is inactive, return immediately. Otherwise advance and draw it via anim_frame_stepper and sprite_dispatcher, skipping the frame advance if the skip counter (frame_counter_hit) is non-zero.
7. Death check: if P1 holds movement 8 or 27, set P2 to movement 19 (victory) with IY=P1 as the dying character. Otherwise set P1 to movement 19 with IY=P2 as the dying character.
8. Goblin sync and exit: when the goblin's X tile reaches 29, deactivate the goblin and select exit animation 29, 31, 32 or 33 based on mirror flag and movement ID.
Movement IDs: 8 = decapitation death, 27 = standard death, 19 = victory pose.
INPUT: None. OUTPUT: None. Framebuffer, character state blocks and fighter_distance updated in place.
DESTROYS: AF, BC, DE, HL, IY, IX.
Frame processing: compute fighter distance, dispatch input or AI for both characters, update animation and X position, draw sprites in Z-order, draw P2 overlay tile, and advance and draw the goblin if active.
combat_loop 9AC1 CALL calc_distance Compute fighter distance and store in fighter_distance.
9AC4 LD IY,$B562 Point IY at P2 state block and dispatch input or AI for P2.
9AC8 CALL input_ai_dispatcher
9ACB CALL char_frame_update Update P2 frame, movement and X position.
9ACE LD IY,$B559 Point IY at P1 state block and dispatch input or AI for P1.
9AD2 CALL input_ai_dispatcher
9AD5 CALL char_frame_update Update P1 frame, movement and X position.
9AD8 LD A,($B55F) Load P1 movement ID and read its properties byte via get_moves_properties.
9ADB CALL get_moves_properties
9ADE BIT 0,A Test draw-order flag (bit 0 of the properties byte).
9AE0 LD IY,$B559 If bit clear, draw P1 then P2 (P2 on top). If bit set, draw P2 only here (P1 draws on top at 9AFA).
9AE4 PUSH IY
9AE6 CALL Z,sprite_dispatcher
9AE9 LD IY,$B562
9AED CALL NZ,sprite_dispatcher
9AF0 CALL Z,sprite_dispatcher
9AF3 PUSH AF Save draw-order flag.
9AF4 CALL p2_overlay_tile Draw P2 overlay tile.
9AF7 POP AF Restore draw-order flag.
9AF8 POP IY Restore P1 state block pointer.
9AFA CALL NZ,sprite_dispatcher Draw-order bit set: draw P1 sprite on top.
9AFD LD IY,$B56B Point IY at the goblin state block.
9B01 LD A,(IY+$00) Load goblin active flag.
9B04 OR A Test if zero.
9B05 RET Z Return if goblin is inactive.
9B06 LD A,($B973) Load frame skip counter.
9B09 OR A Test if non-zero.
9B0A JP NZ,sprite_dispatcher Skip counter active: draw current goblin frame without advancing animation.
9B0D LD A,(IY+$03) Load goblin X tile counter.
9B10 INC A Increment X tile counter.
9B11 AND $1F Wrap to 0-31.
9B13 LD (IY+$03),A Write updated X tile counter back.
9B16 CALL anim_frame_stepper Advance goblin animation frame.
9B19 CALL sprite_dispatcher Draw goblin sprite.
Death check: if P1 holds movement 8 or 27, assign victory (movement 19) to P2 with P1 as the dying character; otherwise assign victory to P1 with P2 as the dying character.
9B1C LD IY,$B559 Point IY at P1 state block for death check.
9B20 LD A,(IY+$06) Load P1 current movement ID.
9B23 CP $08 Is P1 in the decapitation death animation (movement 8)?
9B25 JR Z,combat_loop_0 Yes: take the P1 dying path.
9B27 CP $1B Is P1 in the standard death animation (movement 27)?
9B29 JR Z,combat_loop_0 Yes: take the P1 dying path.
9B2B JR combat_loop_1 P1 not dying: take the P2 dying path.
combat_loop_0 9B2D LD IY,$B562 P1 dying: point IY at P2 and load A=19 for victory.
9B31 LD A,$13
9B33 CALL set_char_moves Apply movement 19 victory to P2 via set_char_moves.
9B36 LD IY,$B559 Point IY back at P1 state block (dying character).
9B3A JR combat_loop_2 Jump to position check.
combat_loop_1 9B3C LD A,$13 P2 dying: load A=19 for victory.
9B3E CALL set_char_moves Apply movement 19 victory to P1 via set_char_moves.
9B41 LD IY,$B562 Point IY at P2 state block (dying character) for position check.
Goblin sync and exit: synchronise goblin step-over with the dying character body, trigger the flying head final phase, and when the goblin X tile reaches 29 deactivate it and select the exit animation based on mirror flag and movement ID.
combat_loop_2 9B45 LD B,(IY+$03) Load dying character's X tile position into B.
9B48 LD A,$07 Load constant 7 for sync offset.
9B4A ADD A,B Add X tile to compute goblin sync threshold (X+7).
9B4B LD B,A Save sync threshold in B.
9B4C LD A,($B56E) Load goblin X tile position from goblin_x_pos.
9B4F CP B Compare goblin X tile with sync threshold.
9B50 JR NZ,combat_loop_3 No match: skip body nudge, jump to head sync check.
Goblin step-over sync fires when goblin_x_pos matches head_screen_addr AND 31 - 1 AND 31. Triggers once as the goblin steps over the landed head, launching it rightward.
9B52 INC (IY+$03) Sync match: nudge dying character X position by 1 for body alignment.
combat_loop_3 9B55 LD B,A Preserve goblin X tile position in B.
9B56 LD A,($B6DF) Load low byte of head framebuffer address from head_screen_addr.
9B59 AND $1F Extract screen column (0-31).
9B5B DEC A Offset by 1 to align with goblin X tracking.
9B5C AND $1F Wrap to 0-31 column range.
9B5E CP B Compare adjusted head column with goblin X tile.
9B5F CALL Z,goblin_kicks Match: trigger flying head final-phase update via goblin_kicks.
9B62 LD A,B Reload goblin X tile position into A.
9B63 CP $1D Test if goblin X tile has reached 29 (body exits the screen).
9B65 RET NZ Not yet: return, fight sequence continues.
9B66 XOR A Goblin X reached 29: deactivate goblin by clearing goblin_state_block.
9B67 LD ($B56B),A
9B6A LD A,(IY+$02) Load the dying character's mirror flag byte.
9B6D BIT 0,A Test bit 0: is the character mirrored (facing left)?
9B6F JR Z,combat_loop_5 Not mirrored: jump to combat_loop_5.
Exit animation selection when mirrored (facing left): movement 27 (death): exit 32 (drag left, mirrored); movement 8 (decapitation): exit 29 (drag left mirrored with decap).
9B71 LD A,(IY+$06) Mirrored: load dying character's movement ID.
9B74 CP $1B Is it movement 27 (standard death)?
9B76 JR Z,combat_loop_4 Yes: jump to combat_loop_4.
9B78 LD A,$1D No (decapitation): apply drag-left (mirrored) with decap exit animation (29) via set_char_moves.
9B7A JP set_char_moves
combat_loop_4 9B7D LD A,$20 Standard death mirrored: apply drag-left (mirrored) exit animation (32) via set_char_moves.
9B7F JP set_char_moves
Exit animation selection when not mirrored (facing right): movement 27 (death): exit 33 (drag right); movement 8 (decapitation): exit 31 (drag right with decap).
combat_loop_5 9B82 LD A,(IY+$06) Not mirrored: load dying character's movement ID.
9B85 CP $1B Is it movement 27 (standard death)?
9B87 JR Z,combat_loop_6 Yes: jump to combat_loop_6.
9B89 LD A,$1F No (decapitation): apply drag-right with decap exit animation (31) via set_char_moves.
9B8B JP set_char_moves
combat_loop_6 9B8E LD A,$21 Standard death not mirrored: apply drag-right exit animation (33) via set_char_moves.
9B90 JP set_char_moves
