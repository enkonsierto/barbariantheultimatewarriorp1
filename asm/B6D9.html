B6D9: Flying head state block.
Initialised by head_init on decapitation (head_state + head_state_flag + head_frame_seq + head_traj_ptr + head_screen_addr).
8-byte block operated via IX by head_animator each frame.
Re-initialised by goblin_kicks via head_reactivate (new tile source + trajectory, direction reset to 0, state=1).
A parallel block at p2_overlay_block_state reuses this same layout for the P2 overlay renderer, allowing render_head_tile and head_mirroring to serve both without modification.
Offset Address Field
IX+0 head_state Direction: 0=right, 1=left
IX+1 head_state_flag State: 0=inactive, 1=bouncing, 255=landed
IX+2,3 head_frame_seq Tile animation sequence pointer (p1_head_tile_seq P1/p2_head_tile_seq P2)
IX+4,5 head_traj_ptr Movement control pointer (normal_traj_data normal/kicked_traj_data after goblin kick)
IX+6,7 head_screen_addr Current framebuffer address
head_state B6D9 DEFB $00 IX+0: Direction flag. 0=right, 1=left. Set by head_init, reset to 0 by goblin_kicks.
head_state_flag B6DA DEFB $00 IX+1: State. 0=inactive (deactivate_head), 1=bouncing (head_init / head_reactivate), 255=landed (mark_head_landed).
head_frame_seq B6DB DEFW p1_head_tile_seq IX+2,3: Tile frame sequence pointer. Advanced +1/frame by head_animator to read the next tile index byte. p1_head_tile_seq player head, p2_head_tile_seq P2 head (normal decap); after goblin kick. Byte 255 triggers landed state via mark_head_landed.
head_traj_ptr B6DD DEFW normal_traj_data IX+4,5: Trajectory data pointer. 2-byte pairs (control_byte, magnitude); advanced +2/frame by head_animator. normal_traj_data normal decapitation, kicked_traj_data after goblin kick.
head_screen_addr B6DF DEFW $0000 IX+6,7: Current framebuffer address. Updated every frame by head_animator, read by shadow_head for shadow placement.
