BA02: AI behaviour profile pointer table.
Two-section block: a 90-entry 2-byte LE pointer table followed immediately by 22 AI profile blocks of 8 bytes each.
SECTION 1 (ai_profile_table): 90 x 2-byte LE pointers indexed by extra_state (p2_frame_index for P1, p1_frame_index for P2). Each pointer targets one of the 22 profile blocks below. Multiple extra_state values share the same profile.
SECTION 2 (ai_profile_0): 22 profile blocks x 8 bytes. Each byte is a movement_block_id (0-22) selecting an 8-byte row in ai_movement_table. The byte used is chosen by second_ai_state_table[fighter_distance] (range 0-7), so the AI state determines which column of the profile is active.
Typical profile structure: first 3 bytes = passive/neutral blocks, middle 3 bytes = aggressive blocks, last 2 bytes = attack-forward blocks (block 4 and 8, both identical: mostly movement-1/attack-5).
Used exclusively by character_ai_engine.
