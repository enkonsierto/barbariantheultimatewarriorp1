93BB: P1-death score update and time drain loop.
Symmetric counterpart to p2_death_score_update. Called by p2_wins_seq (P2-wins sequence) after P1 has been defeated.
MODE 1P/DEMO (timer_flag=255): returns immediately without scoring. The CPU never accumulates a time bonus in 1P mode.
MODE 2P (timer_flag=0..90): awards +10 to P2 score (p2_score) per timer unit drained via timer_tick, until timer_flag reaches 0. Falls through to post_drain_wait implicitly when the loop terminates.
p1_death_score_update 93BB LD A,($B6D8) Read the round timer (timer_flag).
93BE CP $FF Is it 255 (1P/demo mode)?
93C0 RET Z Yes - return without scoring.
p1_death_score_update_0 93C1 LD HL,($B947) HL = current P2 score (p2_score).
93C4 LD DE,$000A DE = 10 points to add.
93C7 ADD HL,DE HL = score + 10.
93C8 LD ($B947),HL Write updated score back to p2_score.
93CB CALL paint_score_0 Refresh P2 score display via paint_score_0.
93CE CALL timer_tick Decrement timer by 1 and update HUD via timer_tick. Returns Z if timer=0.
93D1 JR NZ,p1_death_score_update_0 JTimer not zero - loop back to award another +10. Falls through to post_drain_wait when timer reaches 0.
