8C9C: Single UDG character renderer
Used by the routines at hit_processor, catch_key, draw_word, print_dec_number, energy_dec and game_start.
Draws one 8x8 UDG tile from the font table at udg_font_addr to a VRAM address. Used to render scores, keys glyphs, menu labels and the hit indicator.
Converts the ASCII code to a font index (SUB 48), multiplies by 8 to get the byte offset, and adds the UDG font base to locate the 8-byte tile data. Copies one byte per scanline row, calling calc_next_scanline_addr after each row to advance to the next VRAM scanline.
INPUT: A = ASCII code (48+). HL = VRAM destination address. OUTPUT: None.
DESTROYS: AF, BC, DE, HL.
draw_character 8C9C PUSH HL Preserves the VRAM destination address.
8C9D SUB $30 Converts ASCII code to font index (index = A - 48).
8C9F LD L,A Places font index in HL (L = index, H = 0).
8CA0 LD H,$00
8CA2 ADD HL,HL Multiplies font index by 8 to obtain the byte offset into the font table.
8CA3 ADD HL,HL
8CA4 ADD HL,HL
8CA5 LD DE,$5CA8 Loads the UDG font base address udg_font_addr into DE.
8CA8 ADD HL,DE HL now points to the first byte of the font data for this character.
8CA9 POP DE Restores the VRAM destination address into DE.
8CAA EX DE,HL Swaps registers so HL = VRAM destination, DE = font source.
8CAB LD B,$08 Sets row counter to 8, one iteration per scanline.
draw_character_0 8CAD LD A,(DE) Copies one font scanline byte to the screen.
8CAE LD (HL),A
8CAF CALL calc_next_scanline_addr Advances HL to the next VRAM scanline via calc_next_scanline_addr.
8CB2 INC DE Advances to the next font row.
8CB3 DJNZ draw_character_0 Repeats for all 8 scanlines.
8CB5 RET Return.
