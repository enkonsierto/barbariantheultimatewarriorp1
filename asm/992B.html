992B: Movement transition resolver.
Called from input_gated_moves with the mirror-adjusted key state in A.
Performs a two-level table lookup to find the candidate movement ID:
Level 1: maps the lower 4 direction bits of the key state to an input class (0-8) via input_lookup_table, collapsing 16 combos into 9 meaningful classes.
Level 2: uses the Fire button (IY+1 bit 4) to select the transition table - no_fire_transition_table (no fire: walks, crouches, rolls) or fire_transition_table (fire: attacks) - then resolves a pointer for the current movement ID and reads the resulting movement ID at the input class offset.
Returns 255 if no transition is defined for the current input/movement combination.
INPUT: A = adjusted key state. IY = character state block.
OUTPUT: A = new movement ID, or 255 if no transition applies. DESTROYS: AF, DE, HL.
moves_solver 992B AND $0F Strip the Fire bit; keep only the 4 direction bits.
992D LD HL,$B962 Look up the input class index (0-8) for this direction combination via input_lookup_table.
9930 CALL get_byte_value
9933 LD E,A Store the input class as a 16-bit offset for the pointer walk.
9934 LD D,$00
9936 BIT 4,(IY+$01) Fire pressed - jump to attack transition table at moves_solver_0.
993A JR NZ,moves_solver_0
993C LD HL,$AEFB Fire not pressed - select the movement transition pointer table no_fire_transition_table.
993F JR moves_solver_1 Skip to shared table resolution.
moves_solver_0 9941 LD HL,$AF33 Fire pressed - select the attack transition pointer table fire_transition_table.
moves_solver_1 9944 LD A,(IY+$06) Resolve the 2-byte transition row pointer for the current movement ID via solve_pointer.
9947 CALL solve_pointer
994A ADD HL,DE Advance the pointer by the input class offset to reach the target transition entry.
994B LD A,(HL) Read and return the resulting movement ID.
994C RET
