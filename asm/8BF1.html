8BF1: Human controller input reader.
Called by input_ai_dispatcher with A = controller type (IY+0) and IY pointing to the character state block. For keyboard/Sinclair controllers, reads 5 scancodes (Fire, Up, Down, Left, Right) from the table for this controller type and builds a key-state bitmask.
For Kempston, reads the joystick port directly. Returns A with bits 0-4 set for pressed keys: bit 0 = Right, bit 1 = Left, bit 2 = Down, bit 3 = Up, bit 4 = Fire.
IN A = controller type (1-5). IY = character state block.
OUT A = key state bitmask (bits 0-4). (Caller stores it in IY+1).
DESTROYS AF, BC, DE, HL.
Scancode tables: 1 = P1 keyboard p1_keyboard_table, 2 = P2 keyboard p2_keyboard_table (Sinclair2 defaults), 3 = Sinclair2 fixed sinclair_joystick_2_table (keys 6-0), 4 = Sinclair1 fixed sinclair_joystick_1_table (keys 1-5), 5 = Kempston joystick (port 31).
human_input_reader 8BF1 LD HL,$B949 HL = P1 keyboard scancode table p1_keyboard_table (default for type 1).
8BF4 CP $01 Is controller type = 1?
8BF6 JR Z,human_input_reader_0 Yes: use P1 keyboard table at p1_keyboard_table (jump to scan loop).
8BF8 LD HL,$B950 HL = P2 keyboard scancode table p2_keyboard_table (type 2).
8BFB CP $02 Is controller type = 2?
8BFD JR Z,human_input_reader_0 Yes: use P2 keyboard table at p2_keyboard_table.
8BFF LD HL,$B955 HL = Sinclair 2 fixed scancode table sinclair_joystick_2_table (type 3).
8C02 CP $03 Is controller type = 3?
8C04 JR Z,human_input_reader_0 Yes: use Sinclair 2 table at sinclair_joystick_2_table.
8C06 LD HL,$B95A HL = Sinclair 1 fixed scancode table sinclair_joystick_1_table (type 4).
8C09 CP $04 Is controller type = 4?
8C0B JR Z,human_input_reader_0 Yes: use Sinclair 1 table at sinclair_joystick_1_table.
8C0D CP $05 Is controller type = 5 (Kempston)?
8C0F RET NZ If controller type is not 1-5, return with A unchanged (no input).
8C10 IN A,($1F) Read Kempston joystick state from port 31 into A.
8C12 AND $1F Mask to lower 5 bits (Right, Left, Down, Up, Fire).
8C14 RET Return with Kempston key state in A.
human_input_reader_0 8C15 LD E,$00 E = 0 (result accumulator). D = 5 (remaining keys to scan).
8C17 LD D,$05 Start of keyboard scan loop (5 keys).
human_input_reader_1 8C19 LD A,(HL) A = (HL) read next scancode from table.
8C1A INC HL HL++ advance scancode table pointer.
8C1B CALL key_state_test Test key state via key_state_test: Carry=1 released, Carry=0 pressed.
8C1E CCF CCF invert carry so pressed -> Carry=1.
8C1F RL E RL E rotate carry into E; builds bitmask LSB-first (Right,Left,Down,Up,Fire).
8C21 DEC D D- decrement remaining-key counter.
8C22 JR NZ,human_input_reader_1 Loop until all 5 keys have been processed.
8C24 LD A,E A = E: copy final key-state bitmask to A.
8C25 RET Return with A = key-state bitmask.
