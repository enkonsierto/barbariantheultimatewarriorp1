98E6: Mirror-corrected directional input.
Used by the routines at character_ai_engine and input_gated_moves.
If the character is mirrored (IY+2 bit 0 set) and a horizontal direction is pressed (bits 0 or 1 of A), swaps Right (bit 0) and Left (bit 1) via XOR 3.
This ensures the movement table lookup in moves_solver always treats towards-opponent as Right and away-from-opponent as Left, regardless of which side the character faces.
INPUT: A = key state bitmask (bit 0=Right, bit 1=Left, bit 2=Down, bit 3=Up, bit 4=Fire). IY+2 bit 0 mirror flag.
OUTPUT: A = adjusted key state with Left/Right possibly swapped. DESTROYS: B.
input_mirror_correcter 98E6 LD B,A Save key state; mask to horizontal bits only (AND 3).
98E7 AND $03
98E9 LD A,B No horizontal input pressed; restore A and return unchanged.
98EA RET Z
98EB BIT 0,(IY+$02) Mirror flag clear; return unchanged.
98EF RET Z
98F0 LD A,B Swap Right and Left bits (XOR 3) to correct for mirrored facing.
98F1 XOR $03
98F3 RET Return with adjusted key state in A.
