9A2C: Per-frame hit detection.
Used by the routine at entry_point.
Called once per frame from the main game loop.
Skips entirely if frame_counter_hit is non-zero (throttled by frame_counter_hit_inc).
Runs the attacker/defender check twice: first with P1 as attacker and P2 as defender, then reversed. For each pass, reads the attacker frame index and searches the hit section table hit_detection_table via find_hit_section. On a match, walks
the section pairs comparing defender frame index against IY+8. On a match, calls collision_solver to validate range and resolve the hit.
Hit section table hit_detection_table format: each section starts with the attacker frame ID, followed by (defender frameID, hitTypeIndex) pairs, terminated by 0xFF.
INPUT: None. OUTPUT: None. DESTROYS: AF, BC, DE, HL, IX, IY.
hit_detection 9A2C LD A,($B973) Read frame skip counter frame_counter_hit - return if non-zero.
9A2F OR A
9A30 RET NZ
9A31 LD IX,$B559 Set up P1 as attacker (IX) and P2 as defender (IY).
9A35 LD IY,$B562
9A39 CALL hit_check Call hit_check to run hit detection for this attacker/defender pair.
9A3C LD IY,$B559 Set up P2 as attacker (IX) and P1 as defender (IY).
9A40 LD IX,$B562 Fall through to hit_check for the reversed pass.
hit_check 9A44 LD A,(IX+$08) Load attacker frame index from IX+8.
9A47 LD HL,$AFD7 Search hit section table hit_detection_table for the attacker frame via find_hit_section.
9A4A CALL find_hit_section
9A4D RET NZ Return if attacker frame not found - no active hit this frame.
9A4E PUSH DE Read candidate defender frame ID from the current table pair.
9A4F POP HL
hit_pair_scan 9A50 LD A,(HL)
9A51 CP $FF Compare against defender frame index IY+8.
9A53 RET Z
9A54 CP (IY+$08) Match - call collision_solver to validate range and apply the hit.
9A57 CALL Z,collision_solver Advance to the next pair.
9A5A INC HL Loop until 0xFF sentinel - no more pairs for this attacker frame.
9A5B INC HL
9A5C JR hit_pair_scan
