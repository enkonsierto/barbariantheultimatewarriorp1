8C26: Keyboard scanner: detects single pressed key for input config (redefine keys).
Used by the routine at catch_key.
Scans 8 keyboard rows (port 0xFE) to find exactly one keypress. Used by key assignment (catch_key) to capture scancodes without conflicts.
OUT: D=scancode (0-39) or 255 (none/multiple); Z=0 if valid key.
DESTROYS: AF,BC,DE,H.
keyb_scan 8C26 LD DE,$FF2F D = 255 (no-key sentinel); E = 111b (row/column base counter).
8C29 LD BC,$FEFE B = 0xFE (keyboard row mask); C = 0xFE (ULA keyboard port).
keyb_scan_0 8C2C IN A,(C) IN A,(C) read 5 key bits for this row from port 0xFE.
8C2E CPL CPL invert bits (active-low): 1 = pressed, 0 = released.
8C2F AND $1F AND 31 mask to bits 4-0 (the 5 keys of this row).
8C31 JR Z,keyb_scan_2 If no key bits set in this row, jump to next row.
8C33 INC D First key in this scan: INC D makes D = 0.
8C34 RET NZ If D was already 0, a previous row already had a key -> two keys pressed; abort and return immediately with D=0, Z clear.
8C35 LD H,A H = pressed key bitmask for this row (5 bits).
8C36 LD A,E A = current row/column base (E).
keyb_scan_1 8C37 SUB $08 Subtract 8 repeatedly to align A to the correct row base slot.
8C39 SRL H SRL H shift key bitmask right; lowest set bit falls into carry.
8C3B JR NC,keyb_scan_1 If carry clear, no key at this column; loop to check next column.
8C3D RET NZ Single column found: return now; D already holds scancode.
8C3E LD D,A D = A: store computed scancode (row base + column offset).
keyb_scan_2 8C3F DEC E E- advance row/column base for next row.
8C40 RLC B RLC B rotate row mask left to select next keyboard row.
8C42 JR C,keyb_scan_0 If more rows remain (carry clear), loop back to read next row.
8C44 CP A All rows scanned with no key found: CP A sets Z flag.
8C45 RET Z Return with Z set and D = 255 (no key pressed or conflict).
