AEFB: Movement transition pointer tables.
Two adjacent 28-entry sub-tables, 2 bytes per entry (little-endian), one per movement ID 0-27. Each entry is a pointer to a 9-byte transition row in st_free.
Both tables are indexed by movement ID via solve_pointer, called from moves_solver.
Each pointed row holds one movement ID per input class (0-8) from input_lookup_table. 255 means no transition is defined for that input class.
Multiple movement IDs may share the same row pointer when their transition profiles are identical.
Movement IDs 28-39 are absent: all are either non-interruptible or goblin-only and never reach moves_solver via input_gated_moves.
No-fire table: selected by moves_solver when Fire (IY+1 bit 4) is not pressed. Covers movement IDs 0-27.
