8B81: Input and AI dispatcher.
Called once per character from combat_loop.
IY points to the active character state block. Selects between human input and AI, or returns immediately if the character is inactive. Human-controlled characters read the keyboard/joystick and store the key bitmask in IY+1. CPU-controlled characters call character_ai_engine to generate an AI key bitmask in their state block.
IN IY character state block.
OUT (IY+1) key state bitmask (human or AI).
DESTROYS AF, A. IY is preserved on return.
input_ai_dispatcher 8B81 LD A,(IY+$00) A = (IY+0) controller type / active flag.
8B84 BIT 7,A Test bit 7: 0 = human-controlled, 1 = CPU-controlled.
8B86 JR NZ,input_ai_dispatcher_0 If bit 7 set (CPU), branch to AI path at 35729.
8B88 OR A Human path: A = 0 means inactive character.
8B89 RET Z If inactive (IY+0 = 0), return immediately.
8B8A CALL human_input_reader Call human_input_reader to read human input for this controller type.
8B8D LD (IY+$01),A Store key bitmask returned in A into (IY+1).
8B90 RET Return (human-controlled case done).
input_ai_dispatcher_0 8B91 LD A,IYl CPU path: A = (IY+1) character type (89=P1, 98=P2).
8B93 CP $59 Is this P1 type (89)?
8B95 JR NZ,input_ai_dispatcher_1 If not P1, branch to P2 AI path at 35750.
8B97 LD A,($B56A) Load P1 extra state byte from p2_frame_index (AI profile / difficulty).
8B9A PUSH IY Preserve caller IY.
8B9C LD IY,$B559 Set IY to P1 state block base p1_controller_type.
8BA0 CALL character_ai_engine Call character_ai_engine to run P1 AI and write synthetic key state into IY+1.
8BA3 POP IY Restore original IY.
8BA5 RET Return (CPU-controlled P1 done).
input_ai_dispatcher_1 8BA6 LD A,($B561) P2 AI path: load P2 extra state byte from p1_frame_index.
8BA9 PUSH IY Preserve caller IY.
8BAB LD IY,$B562 Set IY to P2 state block base p2_controller_type.
8BAF CALL character_ai_engine Call character_ai_engine to run P2 AI and write synthetic key state into IY+1.
8BB2 POP IY Restore original IY.
8BB4 RET Return (CPU-controlled P2 done).
