939F: P2-death score update and time drain loop.
Called by p1_wins_seq (P1-wins sequence) after P2 has been defeated.
MODE 1P/DEMO (timer_flag=255): jumps immediately to post_fight_handler end-of-match handler without touching the score. The timer is never decremented.
MODE 2P (timer_flag=0..90): awards 10 points to P1 score (p1_score) and refreshes the score display via paint_score, then calls timer_tick to decrement the timer by 1 and update the HUD. Loops until timer_tick returns Z (timer_flag has reached 0), then falls through to post_drain_wait.
Each iteration awards +10 to P1 and drains 1 timer unit, so the total bonus equals remaining_time * 10.
p2_death_score_update 939F LD A,($B6D8) Read the round timer (timer_flag).
93A2 CP $FF Is it 255 (1P/demo mode - infinite time)?
93A4 JP Z,post_fight_handler Yes - jump to post_fight_handler end-of-match handler without scoring.
p2_death_score_update_0 93A7 LD HL,($B945) HL = current P1 score (p1_score).
93AA LD DE,$000A DE = 10 points to add.
93AD ADD HL,DE HL = score + 10.
93AE LD ($B945),HL Write updated score back to p1_score.
93B1 CALL paint_score Refresh P1 score display via paint_score.
93B4 CALL timer_tick Decrement timer by 1 and update HUD via timer_tick. Returns Z if timer=0.
93B7 JR NZ,p2_death_score_update_0 Timer not zero - loop back to award another +10.
93B9 JR post_drain_wait Timer exhausted - fall through to post_drain_wait.
