95B5: Hit section table search.
Scans the hit section table at HL for an entry whose first byte matches the attacker frame ID in A.
Scans a table of 3-byte entries (frame ID with hit + 2 bytes) until matching A or sentinel = 128. On a match, DE is loaded with the 2-byte data pair from that entry and Z is set.
INPUT: A = attacker frame ID. HL = base address of the hit section table (hit_detection_table or hit_indicator_frame_table).
OUTPUT: Z set on match; DE = 2-byte data pair from the matched entry. DESTROYS: HL, DE, AF, BC.
find_hit_section 95B5 LD D,(HL) Read entry key into D; advance HL to the end of the 3-byte entry.
95B6 INC HL
95B7 INC HL
95B8 INC HL
95B9 BIT 7,D Sentinel reached (128): return NZ (no match).
95BB RET NZ
95BC CP D Frame ID does not match: loop back to next entry.
95BD JR NZ,find_hit_section
95BF DEC HL Match found: backtrack HL and load the 2-byte data pair into DE.
95C0 LD D,(HL)
95C1 DEC HL
95C2 LD E,(HL)
95C3 RET Return with Z set.
