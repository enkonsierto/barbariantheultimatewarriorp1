8C61: Single-key capture prompt.
Used by the routine at key_assign.
Redefining keys, waits for the player to press a key, draws its glyph on screen and plays a confirmation sound. Returns the scancode of the chosen key in A. Loops indefinitely until a valid key press is detected (D != 255).
Called from control_setup in a loop to assign each of the 5 action keys (Fire, Up, Down, Left, Right, and optionally Pause and Quit).
INPUT HL = VRAM address where the key glyph will be drawn.
OUTPUT A = scancode of the selected key.
DESTROYS AF, BC, DE.
catch_key 8C61 PUSH HL Preserve VRAM address.
8C62 LD A,$3F Load ASCII 63 ('?') as the prompt glyph.
8C64 CALL draw_character Draw '?' at HL via draw_character to signal awaiting input.
8C67 CALL waits_112ms Short delay via waits_112ms.
8C6A CALL keyb_scan Scan keyboard via keyb_scan; D = scancode, or 255 if no key pressed.
8C6D POP HL Restore VRAM address.
8C6E LD A,D If D = 255 (no key), loop back to catch_key.
8C6F CP $FF
8C71 JR Z,catch_key
8C73 PUSH AF Save scancode.
8C74 PUSH HL Preserve VRAM address.
8C75 LD HL,$B974 HL = key-glyph lookup table base (key_glyph_table).
8C78 CALL get_byte_value Read display glyph index for this scancode via get_byte_value.
