96F0: Fade ink colour to black.
Used by the routine at menu_fade_out.
For each of C rows, reads the ink bits of each attribute byte via AND 7 and decrements the ink value by 1 unless it is already 0. Processes B tiles per row, then advances HL 32 bytes to the next attribute row.
Called from menu_fade_out to progressively darken the game window during the fade-out transition.
INPUT: HL = first attribute address (top-left tile). B = tiles per row (26). C row count (3).
OUTPUT: None. DESTROYS A, C, DE, HL.
fade_ink 96F0 PUSH HL Save row start address and counters.
96F1 PUSH BC
fade_ink_0 96F2 LD A,(HL) Read ink bits from current attribute (AND 7).
96F3 AND $07
96F5 OR A Ink already 0 - skip decrement.
96F6 JR Z,fade_ink_1
96F8 LD A,(HL) Reload attribute and decrement ink by 1.
96F9 DEC A
96FA LD (HL),A Write decremented value back to attribute byte.
fade_ink_1 96FB INC HL Advance to next tile in row.
96FC DJNZ fade_ink_0 Repeat for all B tiles in this row.
96FE POP BC Restore counters and row start address.
96FF POP HL

9705 JR NZ,fade_ink
